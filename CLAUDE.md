## Структура данных
```
DB → objects[] → stages[] → positions[] → works[] + mats[]
```
- **Object**: id, name, icon (emoji), totalBudget, stages[]
- **Stage**: id, name, icon, color, status, positions[], endDate, note
- **Position**: id, name, status ('idle'|'active'|'done'), works[], mats[]
- **Work**: id, name, contractAmount, paid, contractor, phone, startDate, endDate, note, done
- **Mat**: id, name, qty, unit, price, note

## Калькулятор — комнатная модель «По квартире» (tabs.js)
**Пол = система «финиш + основание»**, а не просто покрытие (инсайт Дмитрия 10 июня).
- **Состояние:** `let _aRooms=[{id,name,L,W,floor}]` — список комнат, у каждой свой тип пола. `_aEnsureRooms/_aAddRoom/_aDelRoom/_aRoomName/_aRoomDim/_aRoomFloor` управляют. Размеры (Д×Ш), а не площадь — нужны для стен (периметр), внутренних стен и чтения плана ИИ.
- **`FLOOR_SYSTEMS`** (расширяемый объект) — каждый тип пола тянет рецепт материалов от площади: `ceramic` (стяжка→клей→затирка→керамогранит), `laminate` (стяжка→подложка→ламинат), `plank` (лаги→чёрный пол→доска→лак, **без стяжки**), `screed`. Строка материала `[ключ_цены, ru, az, en, qty(a), ед]`. Добавить кварцвинил/линолеум/паркет/размеры плитки — сюда же, без переделки.
- **`calcUpdate` apart-ветка:** агрегирует площади по типу пола (`areaByType`), считает рецепт на ОБЩУЮ площадь типа (мешки округляются один раз), суммирует одинаковые материалы из разных комнат в одну строку (`agg`). Стены/потолок — площадь из суммы комнат (Σ периметров × высота; Σ площадей), отделка общая на квартиру. Итог: пол+стены+потолок, кнопка `createApartStages` → `_applyApartStages` (пол из `floorAgg`).
- **Прайс расширен** (`_PRICES_DEF` в data.js + `showPriceBook`/`savePricesUI` в tabs.js): добавлены glue, screed, tileAdh, grout, tile, underlay, laminat, lag, plywood, plank, lacquer, paint — теперь система считает деньги, а не только количества.
- **Связка на будущее:** комнаты с размерами → внутренние стены `≈ (Σ периметров комнат − наружный периметр)/2` → внутренняя лента фундамента (для «По дому»); и ровно та структура, что ИИ вытащит из плана. Комнатная модель — ядро под обе фичи.
- ⚠️ Мёртвый код после переделки: старые `_aFloorMat/_aScreedTh/_aLamWst/_aTileWst` + селекторы `_aFloorSel/_aScreedThSel/...` больше не вызываются (пол теперь per-room). Не удалял (globals на минифицированной строке 1).
- ⏳ Не сделано в этот заход (отложено Дмитрием «дальше подумаем»): внутренние стены «По дому» + лента; стены/потолок по комнатам; кварцвинил/линолеум/паркет/размеры плитки; дизайн-причёска picker'а режимов.
