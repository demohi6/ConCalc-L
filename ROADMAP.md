# Tikinti — Roadmap

## Цели проекта

### 1. UX — Понятность и простота
- Приложение должно быть интуитивно понятно любому строителю без обучения
- Минимум кликов для основных действий
- Чистый и профессиональный дизайн

### 2. База подрядчиков
- Все подрядчики которые когда-либо добавлялись сохраняются в базе
- Автозаполнение при вводе имени/телефона
- Профиль подрядчика: имя, телефон, специализация, история работ

### 3. Функционал уровня Microsoft Project
- Роли пользователей:
  - **Хозяин** — полный доступ, финансы, все объекты
  - **Прораб** — управление этапами и работами, без финансов
  - **Мастер** — только свои работы и задачи
  - **Бухгалтер** — только финансовая часть
  - **Продавец материалов** — каталог и поставки
- Диаграмма Ганта
- Зависимости между задачами
- Уведомления и напоминания

### 4. Аккаунты и облако
- Регистрация/вход по email или телефону
- Синхронизация данных через сервер (Firebase или собственный)
- Несколько пользователей на один объект
- Офлайн режим с синхронизацией при подключении

### 5. Рейтинг и маркетплейс
- База мастеров и бригад с рейтингом
- База продавцов строительных материалов
- База строительных компаний
- Отзывы и история работ

### 6. Кроссплатформ
- PWA — уже работает (iOS, Android, Desktop)
- Оптимизированный десктоп интерфейс для бухгалтера
- Возможно нативные приложения в будущем
